# Magnes Studio - 需求调研报告

## 1. 项目背景

- **业务驱动**：为小红书内容创作者、品牌电商运营团队和内容工作室提供 AI 驱动的图文内容生产工作台，降低内容策划与设计成本，提升批量生产效率。
- **现状痛点**：
  - 小红书内容生产高度依赖人工：文案撰写、背景设计、排版合成需多人协作，周期长。
  - AI 生成工具与设计工具割裂：文案工具、图像工具、排版工具各自独立，缺乏统一工作流。
  - 品牌一致性难以保障：多人协作时风格不统一，缺乏可复用的风格模版管理。
  - 国内部署受限：部分生图 API（如 Nano-banana）调用链路复杂，缺乏统一代理层。
- **核心诉求**：在统一的可视化画布上，支持从灵感输入到内容生成、排版合成、一键导出的完整 AI 工作流，支持 RAG 知识库驱动的风格记忆，可在国内环境稳定部署。

## 2. 调研方法与参与方

| 角色 | 需求关注点 |
| ---- | ---------- |
| 小红书内容运营 | 快速批量生产高质量图文，风格可复用，支持批量审核与导出 |
| 品牌设计师 | 生成结果符合品牌调性，支持风格参考图反推 |
| 电商商品运营 | 商品主图快速生成，背景可替换，文案与图片协同生成 |
| 技术负责人 | 系统可扩展、多 LLM Provider 支持、私有化部署可行 |
| 内容合规审核 | 敏感词过滤、生成内容可追溯、历史记录可审计 |

调研手段包括：内部用户访谈、竞品分析（Canva AI、创客贴 AI、即梦 AI）、对现有生成工具的可用性测试与内容生产流程走查。

## 3. 竞品与基准分析

| 方案 | 优势 | 局限 |
| ---- | ---- | ---- |
| 可画 AI | 模版丰富，生成质量高 | 无法与本地知识库集成，不支持 Agent 工作流 |
| 创客贴 AI | 国内可用，小红书场景适配好 | AI 深度有限，无多智能体协作，工作流不可编程 |
| 即梦 AI（字节） | 生图质量优秀，国内可用 | 单一生图工具，缺乏文案+排版+导出的一体化能力 |
| ComfyUI + 自研插件 | 高度可定制 | 技术门槛高，非技术用户无法使用，部署复杂 |

**定位**：构建面向内容创作团队的 AI 工作流平台，以 ReactFlow 可视化画布为交互中心，多智能体系统为生产引擎，RAG 知识库为品牌记忆基础，提供从策划到导出的一站式能力。

## 4. 业务目标

1. **效率目标**：单条小红书图文内容从需求到可导出成品，端到端时间 ≤ 5 分钟；支持批量生产小红书内容图片。
2. **质量目标**：生成文案覆盖标题、正文、话题标签；生成图片支持风格一致性控制；排版合成支持多种预设模版；最终输出支持海报级分辨率导出。
3. **合规目标**：内置敏感词检测，所有生成内容可追溯（存储生成历史），支持人工审核与二次编辑。
4. **运营目标**：支持多 LLM Provider（OpenAI 兼容接口、自定义模型），私有化部署可行，具备健康检查、日志、生成历史审计体系。
5. **协作目标**：支持用户账号体系，实现内容创作的多人协作与权限管理。
6. **体验目标**：提供精细编排能力，支持图层级别的可视化编辑、撤销重做、字体切换，降低内容微调成本。

## 5. 外部接口与法规约束

- **Nano-Banana 2**：生图服务，需合规使用，禁止生成违禁内容。
- **Qwen 视觉模型（通义千问）**：图层分析与风格反推，需遵守阿里云服务条款。
- **MCP（Model Context Protocol）**：外部工具调用协议，需控制调用权限与频率。
- **ChromaDB + LlamaIndex**：本地向量数据库，数据存储在本机，无外部数据传输；需定期备份。
- **隐私**：不存储用户个人身份信息；生成历史仅包含业务内容，存储于本地 SQLite；支持配置数据保留策略。

## 6. 用户画像与使用场景

| Persona | 需求亮点 | 场景示例 |
| ------- | -------- | -------- |
| 小红书运营 | 快速批量内容生产、风格统一、支持对话式调整 | 打开 Magnes Studio，拖入商品参考图，对话描述风格，一键生成 10 条图文内容并导出 |
| 品牌设计师 | 风格参考图反推、高分辨率导出 | 上传品牌视觉规范图，Refiner 节点提取风格参数，Painter 生成背景，导出 2160×2880 海报 |
| 电商商品运营 | 商品图智能分层、背景替换、主图快速生成 | 上传白底商品图，Slicer 节点提取商品主体，Painter 生成场景背景，Composer 合成主图 |
| 内容审核专员 | 查看生成历史、敏感词检测结果、人工干预 | 在历史记录面板查看所有生成任务，核查文案合规性，必要时手动编辑后重新导出 |

## 7. 关键业务成功要素

1. **多智能体协同**：Planner（对话意图）、Slicer（图层分析）、Painter（背景生成）、Refiner（风格反推）、CopyWriter（文案生成）、Composer（排版合成）相互配合，覆盖完整内容生产链路。
2. **RAG 知识库驱动**：ChromaDB + BM25 混合检索，积累品牌风格记忆与灵感素材，提升生成一致性。
3. **灵活模版系统**：支持自定义排版模版，存储于 SQLite，可复用与分享。
4. **SSE 实时流式反馈**：生成过程实时可见，用户无需等待黑箱完成，体验流畅。
5. **可视化工作流画布**：基于 ReactFlow，支持拖拽节点、连线、并行分支，非技术用户也可构建工作流。
6. **部署易用性**：前端为纯静态文件，后端为 Python FastAPI，支持本机直接运行，也可容器化部署。

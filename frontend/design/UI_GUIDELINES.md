# Magnes Studio UI 设计规范
本规范旨在定义 Magnes Studio 的视觉标准与组件开发指南，确保所有新开发的 UI 组件在风格、比例及交互体验上保持高度一致。



## 1. 核心视觉理念

* **专业与洗练**：移除多余的装饰词（如英文后缀、表情符号），采用纯净、直观的中文表达。

* **一致性缩放**：所有 UI 元素在 1x 画布缩放下必须清晰易读。

* **层级分明**：通过阴影、边框和实色背景区分节点层级，摒弃过度透明的设计。



## 2. 布局与尺寸规范 (Layout)

### 2.1 节点宽度 (Node Width)

* **业务逻辑节点** (如：内容输入、风格实验、AI绘图、预览)：统一为 **`320px`**。

* **基础原子节点** (如：BaseNode 默认)：统一为 **`155px`**。



### 2.2 间距 (Padding)

* **全局内容间距**：所有节点主体的内边距统一使用 **`p-3` (12px)**。

* **组件内部间距**：优先使用 `gap-2` 或 `gap-3` 维持元素间的呼吸感。



### 2.3 容器比例 (Aspect Ratio)

* **1:1 (Square)**：图片上传框、预览显示区域。

* **4:3**：批量文案输入框、AI 绘图描述区域。

* **3:4**：移动端海报模板预览。



## 3. 字体规范 (Typography)

* **标准字体大小**：所有节点内的正文、标签、提示文字、按钮文字统一为 **`12px`**。

* **最小字体限制**：严禁使用低于 `12px` 的字体，以保证在画布缩放下可读。

* **字体样式**：

* 标题：`font-bold`

* 正文：`font-medium` 或 `font-normal`

* 输入框提示词：`text-slate-400`



## 4. 交互与密度规范 (Interactions)

### 4.1 紧凑型设计 (Compact Design)

* **高密度菜单**：侧边栏菜单、下拉菜单应保持紧凑，垂直间距（py）不宜过大。单个条目高度应控制在能够容纳 12px 文字及图标的最小空间。

* **控制项留白限制**：按钮、选择器之间的间距（gap）应保持在 1-1.5 之间。严禁过度留白，确保在 1x 画布下能一次性展示更多操作选项。

* **按钮规格**：操作按钮（如：复制、画布、对话）统一采用紧凑布局，内部文字与图标的 py 间距应控制在 2 (8px) 以内。



## 5. 图标规范 (Icons)

* **核心组件库**：全面使用 **Lucide Icons**。

* **图标尺寸对比**：

* **空状态/占位图标**：统一为 **`size: 40`**，颜色为 `text-slate-300`。

* **交互按钮图标** (如删除、上传)：统一为 **`size: 16`**。

* **Header/标题图标**：统一为 **`size: 12`**。



## 5. 色彩系统 (Color Palette)

* **节点主题色**：

* 内容输入：`bg-amber-500`

* 风格实验：`bg-cyan-500`

* 手机预览：`bg-indigo-500`

* AI 绘图：`bg-purple-600`

* **背景色设计**：

* 节点主背景：`bg-white` (不透明，实色)。

* 输入框/背景块：`bg-slate-50/50` 或 `bg-gray-50`。

* 边框色：默认 `border-slate-200`，悬浮/选中 `border-blue-500`。



## 6. 组件结构限制

* **BaseNode 依赖**：所有 React Flow 节点必须基于 `BaseNode` 封装，确保阴影、圆角和连接点的一致。

* **禁止冗余边框**：内部组件如输入框，应避免使用深色或过厚的边框，保持轻量视觉感。

* **文案净化**：严禁在 UI 标签中添加 `(Rednote)` 等冗余英文标注，保持界面纯闭环。

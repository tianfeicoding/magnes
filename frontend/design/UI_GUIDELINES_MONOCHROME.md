# Magnes Studio UI 设计规范 (Precision Monochrome Edition)

本规范是 Magnes Studio 的高级视觉分支，旨在实现“建筑级精密” (Architectural Precision) 的单色视觉美学。该风格融合了瑞士平面设计、工程图纸以及现代终端界面的直观性与严谨性。

---

## 1. 核心设计哲学 (Design Philosophy)
*   **工业图纸级美学 (Blueprint Aesthetic)**：界面应像一份精密施工图，不仅展示功能，更展现逻辑结构。
*   **微精密 (Micro-Precision)**：弃用冗余的装饰，利用“细线 (Hairlines)”和“重线 (Heavy Lines)”的对比来建立信息深度。
*   **绝对直角 (Zero Radius)**：全界面 0px 圆角。严禁圆角，以契合硬核工业与精密排版的基调。
*   **视觉平衡 (Visual Equilibrium)**：即使在高密度布局下，也要通过严格的边框对齐和连体布局实现稳定的视觉重心。

## 2. 边框与线条规范 (Border & Line Discipline)
线条是本系统的核心骨架，需严格区分权重：
*   **标准细线 (1px / border-black)**：
    *   **用途**：全场通用。包括节点外框、基础按钮、按钮组、输入框底线、预览区边框。
    *   **核心原则**：**全场唯一权重**。严禁出现 2px 或更粗的线条。
    *   **线条样式**：**全程实线 (Solid)**。严禁在上传区域、占位符或容器中使用 `dashed` (虚线) 或 `dotted` (点线)。所有容器边界必须清晰、稳重。

## 3. 标准化组件布局 (Component Standards)
### 3.1 按钮与操作组 (Buttons & Action Groups)
*   **基础按钮 (Standard Button)**：1px 黑边，0px 圆角。
*   **连体式操作组 (Connected Groups)**：
    *   **应用场景**：节点底部的核心功能触发（如预览、微调、导出等主要操作）。
    *   **实现方式**：使用 `flex -space-x-[1px]`。多个按钮物理连接，共享中间边框。
    *   **布局协议**：必须移除左右内边距，宽度与容器 1:1 齐平。
*   **按钮状态规范 (Button States)**：
    *   **默认 (Default)**：白底黑字黑边 / 黑底白字（主要操作）。
    *   **悬停 (Hover)**：反相显示或轻微灰色背景。
    *   **禁用 (Disabled)**：
        *   **视觉**：`bg-zinc-200` (背景) + `text-zinc-500` (文字) + `border-zinc-200` (无边框)。
        *   **交互**：`cursor-not-allowed`。

### 3.2 极简下拉菜单 (Underline Selects)
*   **实现方式**：仅保留底部边框 (`border-b border-black`)。
*   **布局**：水平并列排放，中间保持较大间距 (`gap-4`)。
*   **背景**：背景透明，去除灰色填充或完整方框。

### 3.3 无界输入框 (Borderless Inputs)
*   **布局**：整体采用 4:3 或 1:1 比例区域。
*   **边框**：移除四周闭合边框，改用 **1px 底部黑线**。
*   **文字**：内部占位符使用斜体衬线字体 (font-serif italic)，输入正文使用等宽字体 (Geist Mono)。

### 3.4 统一空状态 (Standardized Empty States)
*   **核心色彩**：统一使用 `zinc-500` (深灰色)，避免使用纯黑或浅灰，以维持 UI 的通透感与精密感。
*   **图标规范**：
    *   **尺寸**：统一为 32px (在排版紧凑区域可微调为 30px)。
    *   **粗细**：强制指定 1单位 (Stroke Width = 1)，确保线条纤细、具有工业蓝图般的精密感。
*   **文字规范**：
    *   **字号**：统一为 12px，加粗 (font-black)。
    *   **语言**：**全中文对齐**。禁止中英混排，禁止出现 "AWAITING DATA" 等英文占位符。
    *   **间距**：图标与文字之间使用 `gap-3`。
*   **边框样式**：优先使用 `border-black` 实线，严禁使用 dashed (虚线) 以维持视觉稳定。

## 4. 空间与内边距 (Spatial Logic)
*   **标准内边距协议 (Standard Padding Protocol)**：
    *   节点内的主要内容区域（如输入框、风格按钮云、色觉控制区）应**统一使用标准内边距 (p-2)**。
    *   这是组件的基础内边距标准，旨在**确保精密排版的通透感**，使内容与边框间保持稳定的 12px (p-2) 呼吸空间。
    *   **单一层级原则 (Single-Layer Rule)**：内边距应由最外层容器 (`BaseNode`) 统一提供。内嵌子组件严禁再次声明内边距（如 `p-2`），以防止边距叠加出现“套娃”感，维持视觉上的紧凑与精密。
*   **页眉间距**：页眉 (Header) 和底部连体按钮区维持原有布局协议，确保整体比例协调。
*   **最小字号**：全界面严禁出现小于 **12px** 的文字，确保在高精度窄边框下仍具备生产力级别的可读性。

## 5. 字体与颜色 (Typography & Color)
*   **单色调色盘**：
    - `#000000` (100% Black)：**唯一正文与边框色**。不论重要性级别，正文和线条严禁使用灰色。
    - `#FFFFFF` (100% White)：**唯一背景色**。移除 `#F8FAFC` 等背景灰，确保画面的极致通透。
    - `#A1A1AA` (Zinc-400)：用于输入框占位符 (Placeholder)。
    - `#E4E4E7` / `#71717A`：禁用态专用灰阶（背景、文字、边框全灰）。
    *   `Opacity 40%`：用于次要标签（如“尝试：”、“配置：”）。
*   **字体系列**：
    *   **中文字体**：苹方 / 思源黑体 (Bold & Normal)。
    *   **西文字体**：JetBrains Mono (程序感) / Playfair Display (艺术气质标题)。
